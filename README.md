# lingua-manage-crm

**lingua-manage-crm** — это комплексная веб-платформа (CRM/LMS) для автоматизации работы репетитора английского языка и организации учебного процесса студентов. Система поддерживает разделение ролей (Преподаватель / Студент) и предоставляет полный набор инструментов для ведения расписания, контроля успеваемости и обмена учебными материалами.

Проект разработан в качестве выпускной дипломной работы в Компьютерной Академии ТОП.

## 👥 Ролевая модель и функционал

### 👨‍🏫 Кабинет Преподавателя (Администратор)
- **Авторизация (Вход):** Доступ к личному кабинету по учетным данным.
- **Личный профиль:** Просмотр своего профиля.
- **Управление студентами:** Добавление новых студентов (для их входа), просмотр детального профиля (уровень языка, статус обучения «Активен/Завершен»).
- **Интерактивный Календарь занятий (Full CRUD):** Создание, просмотр, редактирование и удаление уроков в сетке расписания.
- **Менеджер учебных материалов:** Публикация и удаление полезных ссылок на учебники, аудиоматериалы и тесты для студентов.
- **Личные заметки (To-Do):** Управление личными рабочими заметками и планами на студентов (создание, удаление, отметка «выполнено»).

### 🧑‍🎓 Кабинет Студента
- **Авторизация (Вход):** Доступ к личному кабинету по учетным данным, созданным преподавателем.
- **Личный профиль:** Просмотр своего профиля.
- **Просмотр расписания:** Доступ к календарю занятий, добавленных репетитором.
- **Библиотека материалов:** Просмотр учебных материалов, прикрепленных преподавателем в виде ссылок.
- **Личные заметки (To-Do):** Создание и ведение собственного списка задач (записать слова, повторить правила) с возможностью отмечать выполненное.

## 🚀 Технические особенности реализации

- **Multi-role Routing:** Защищенные маршруты на базе `React Router v7`, разграничивающие доступ к панели преподавателя и кабинету студента.
- **Global State Management:** Zustand хранит единый источник правды для расписания, списков студентов, заметок и ссылок.
- **Persistent Storage:** Все CRUD-операции, изменения статусов и добавленные материалы автоматически синхронизируются и сохраняются в `LocalStorage` через Zustand Middleware.
- **TypeScript Data Models:** Строгая типизация всех сущностей приложения (интерфейсы для `User`, `Lesson`, `Note`, `Material`).
- **Interactive UI Components:** Использование модальных окон для создания/редактирования записей и интерактивных фильтров для работы с данными.

## 🛠 Технологический стек

- **Frontend:** React 
- **Язык разработки:** TypeScript
- **State Management:** Zustand 
- **Маршрутизация:** React Router v7
- **Стилизация:** Tailwind CSS / Ant Design (для интерактивных таблиц и календаря)
- **Сборщик проекта:** Vite
